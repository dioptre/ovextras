Detailed Description

manages the texture resources used by glGPictureSymbol. The class is slightly different from the OpenGLDListManager and OpenGLVBOManager as it takes a string in the constructor

Constructor & Destructor Documentation

OpenViBEApplications::OpenGLTextureManager::OpenGLTextureManager ( OpenViBE::CString  sourceFile)

The string sourceFile will be used to see if a texture for this file already exist (saved in m_mSourceFile2IdMap) A resource id will be created as well for this texture and will be mapped to its file name by m_mId2SourceFileMap

Parameters:
sourceFilethe file name of the png file that will serve as texture
